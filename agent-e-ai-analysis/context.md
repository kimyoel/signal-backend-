### 웹 프레임워크: FastAPI 선택
- **결정**: FastAPI 사용
- **이유**: 
  - Python에서 가장 빠른 비동기 웹 프레임워크
  - asyncio 기반이라 3개 AI를 동시에 호출하기 최적
  - 자동 API 문서 생성 (Swagger UI)
  - 마스터 설계서에서 Python + FastAPI 지정
- **탈락한 대안**: Flask (이유: 비동기 지원 부족, 병렬 호출에 불리)

### AI 모델 선택
- **결정**: GPT-4o / Gemini 1.5 Flash / Grok-2 / Claude 3 Haiku
- **이유**:
  - GPT-4o: 거시경제 분석에 가장 정확, 범용성 높음
  - Gemini 1.5 Flash: 비용 저렴 ($0.075/1M tokens), 데이터 분석에 적합
  - Grok-2: xAI의 소셜미디어 데이터 접근 가능, 심리 분석에 적합
  - Claude Haiku: 빠르고 저렴, 검증(필터링) 용도로 충분
- **참고**: 마스터 설계서 기준 모델명과 실제 API 사용 모델이 다를 수 있음 (GPT-5.2→GPT-4o 등). 실제 사용 가능한 최신 모델로 적용.

### Grok API 호출 방식
- **결정**: OpenAI 호환 API 형식 사용 (openai 패키지 + base_url 변경)
- **이유**: xAI가 OpenAI와 동일한 API 형식을 제공하므로 별도 라이브러리 불필요

### 캐시 전략
- **결정**: Supabase ai_analyses 테이블에 24시간 TTL 캐시
- **이유**: 
  - 같은 뉴스를 여러 사용자가 분석 요청할 확률 높음
  - 별도 Redis 없이 기존 DB로 처리 → 비용/복잡도 절감
  - 캐시 히트율 70%+ 예상 → 실질 AI API 비용 30% 수준으로 절감
- **탈락한 대안**: Redis (이유: 추가 인프라 비용, MVP 단계에서 과잉)

### 배포 환경
- **결정**: Railway
- **이유**: 마스터 설계서 지정. Python 상시 실행 서버에 적합. 환경변수 관리 편리.

### 크레딧 시스템
- **결정**: users 테이블의 ai_credits 컬럼으로 관리
  - FREE: 5 (매월 리셋)
  - BASIC/PRO: -1 (무제한)
- **이유**: DB 스키마 문서(01_db_schema.md) 기준 그대로 따름
